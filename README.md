# TripPocket

Мобильное приложение для учёта наличных трат в путешествиях: записываешь покупку
в пару тапов (сумма + валюта + категория), а приложение считает итоги, бюджет и
аналитику. Работает офлайн, данные хранятся локально на телефоне в SQLite.

> Технологии: React Native (Expo) · expo-router · expo-sqlite · TypeScript ·
> i18n (RU/EN) · gifted-charts. Подробная архитектура и план — в `claude.md`.

## Возможности

- 🧳 **Поездки** с бюджетом и базовой валютой
- ⚡ **Быстрый ввод траты**: сумма, валюта, категория, комментарий
- 💱 **Мультивалютность** с авто-курсами (онлайн, с кэшем для офлайна)
- 📊 **Аналитика**: итого/бюджет/остаток, по категориям (пончик), по дням
  (график), среднее в день и прогноз
- 🏷️ **Категории**: готовый набор + свои
- 🌐 **Два языка** (русский / английский) с переключением
- 📤 **Экспорт** данных в CSV и JSON-бэкап

## Запуск на iPhone (через Expo Go)

Самый простой способ — без Apple Developer аккаунта и Xcode:

1. Установи **Expo Go** из App Store на iPhone.
2. В папке проекта запусти dev-сервер:
   ```bash
   npm install      # один раз
   npx expo start
   ```
3. Наведи **камеру iPhone** на QR-код в терминале → тапни по уведомлению.

> Телефон и компьютер должны быть в одной Wi-Fi сети. Если QR не сработал —
> открой Expo Go и введи адрес `exp://…` из терминала вручную.

## Запуск в эмуляторе

```bash
npx expo start --ios       # iOS-симулятор (нужен Xcode)
npx expo start --android   # Android-эмулятор
```

## Где хранятся данные

Локально на устройстве в SQLite (`trippocket.db`). База создаётся автоматически
при первом запуске и развивается через миграции (`PRAGMA user_version`) — обновление
кода данные не теряет. Для переустановки/смены телефона используй экспорт
(Настройки → Экспорт данных).

## Структура проекта

```
app/                 экраны (expo-router)
  (tabs)/            табы: добавить, траты, аналитика, поездки, настройки
  trip/new.tsx       создание поездки (модалка)
  expense/[id].tsx   правка траты (модалка)
  categories.tsx     управление категориями (модалка)
src/
  db/                SQLite, миграции, типы
  repositories/      слой доступа к данным (без SQL в экранах)
  services/          валюты, аналитика, экспорт
  hooks/             загрузка данных
  components/        переиспользуемый UI
  i18n/              переводы RU/EN
  lib/               валюты, даты, id, категории
  theme/             дизайн-токены
```
